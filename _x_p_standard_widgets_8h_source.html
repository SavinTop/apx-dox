13/*
14 * XPStandardWidgets - THEORY OF OPERATION
15 *
16 * The standard widgets are widgets built into the widgets library. While you
17 * can gain access to the widget function that drives them, you generally use
18 * them by calling XPCreateWidget and then listening for special messages,
19 * etc.
20 *
21 * The standard widgets often send mesages to themselves when the user
22 * performs an event; these messages are sent up the widget hierarchy until
23 * they are handled. So you can add a widget proc directly to a push button
24 * (for example) to intercept the message when it is clicked, or you can put
25 * one widget proc on a window for all of the push buttons in the window. Most
26 * of these messages contain the original widget ID as a parameter so you can
27 * know which widget is messaging no matter who it is sent to.
28 *
29 */
30

140/***************************************************************************
141 * BUTTON
142 ***************************************************************************/
143/*
144 * The button class provides a number of different button styles and
145 * behaviors, including push buttons, radio buttons, check boxes, etc. The
146 * button label appears on or next to the button depending on the button's
147 * appearance, or type.
148 *
149 * The button's behavior is a separate property that dictates who it hilights
150 * and what kinds of messages it sends. Since behavior and type are different,
151 * you can do strange things like make check boxes that act as push buttons or
152 * push buttons with radio button behavior.
153 *
154 * In X-Plane 6 there were no check box graphics. The result is the following
155 * behavior: in x-plane 6 all check box and radio buttons are round
156 * (radio-button style) buttons; in X-Plane 7 they are all square (check-box
157 * style) buttons. In a future version of x-plane, the xpButtonBehavior enums
158 * will provide the correct graphic (check box or radio button) giving the
159 * expected result.
160 *
161 */
162

269/*
270 * The text field widget provides an editable text field including mouse
271 * selection and keyboard navigation. The contents of the text field are its
272 * descriptor. (The descriptor changes as the user types.)
273 *
274 * The text field can have a number of types, that effect the visual layout of
275 * the text field. The text field sends messages to itself so you may control
276 * its behavior.

278 * If you need to filter keystrokes, add a new handler and intercept the key
