71/*
72 * XPLMFlightLoop_f
73 *
74 * This is your flight loop callback. Each time the flight loop is iterated
75 * through, you receive this call at the end. You receive a time since you
76 * were last called and a time since the last loop, as well as a loop counter.
77 * The 'phase' parameter is deprecated and should be ignored.
78 *
79 * Your return value controls when you will next be called. Return 0 to stop
80 * receiving callbacks. Pass a positive number to specify how many seconds
81 * until the next callback. (You will be called at or after this time, not
82 * before.) Pass a negative number to specify how many loops must go by until
83 * you are called. For example, -1.0 means call me the very next loop. Try to
84 * run your flight loop as infrequently as is practical, and suspend it (using
85 * return value 0) when you do not need it; lots of flight loop callbacks that
86 * do nothing lowers x-plane's frame rate.
87 *
88 * Your callback will NOT be unregistered if you return 0; it will merely be
89 * inactive.
90 *
91 * The reference constant you passed to your loop is passed back to you.
92 *
93 */
94typedef float (* XPLMFlightLoop_f)(
95 float inElapsedSinceLastCall,
96 float inElapsedTimeSinceLastFlightLoop,
97 int inCounter,
98 void * inRefcon);

208/*
209 * XPLMScheduleFlightLoop
210 *
211 * This routine schedules a flight loop callback for future execution. If
212 * inInterval is negative, it is run in a certain number of frames based on
213 * the absolute value of the input. If the interval is positive, it is a
214 * duration in seconds.
215 *
216 * If inRelativeToNow is true, ties are interpretted relative to the time this
217 * routine is called; otherwise they are relative to the last call time or the
218 * time the flight loop was registered (if never called).
219 *
220 * THREAD SAFETY: it is legal to call this routine from any thread under the
221 * following conditions:
222 *
223 * 1. The call must be between the beginning of an XPLMEnable and the end of
224 * an XPLMDisable sequence. (That is, you must not call this routine from
225 * thread activity when your plugin was supposed to be disabled. Since plugins
226 * are only enabled while loaded, this also implies you cannot run this
227 * routine outside an XPLMStart/XPLMStop sequence.)
228 *
229 * 2. You may not call this routine re-entrantly for a single flight loop ID.
230 * (That is, you can't enable from multiple threads at the same time.)
231 *
232 * 3. You must call this routine between the time after XPLMCreateFlightLoop
233 * returns a value and the time you call XPLMDestroyFlightLoop. (That is, you
234 * must ensure that your threaded activity is within the life of the object.
235 * The SDK does not check this for you, nor does it synchronize destruction of
236 * the object.)
237 *
238 * 4. The object must be unscheduled if this routine is to be called from a
239 * thread other than the main thread.
240 *
241 */
242XPLM_API void XPLMScheduleFlightLoop(

244 float inInterval,
