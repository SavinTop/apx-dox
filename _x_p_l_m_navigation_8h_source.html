13/*
14 * XPLMNavigation - THEORY OF OPERATION
15 *
16 * The XPLM Navigation APIs give you some access to the navigation databases
17 * inside X-Plane. X-Plane stores all navigation information in RAM, so by
18 * using these APIs you can gain access to most information without having to
19 * go to disk or parse the files yourself.
20 *
21 * You can also use this API to program the FMS. You must use the navigation
22 * APIs to find the nav-aids you want to program into the FMS, since the FMS
23 * is powered internally by x-plane's navigation database.
24 *
25 */
26

87/*
88 * XPLMNavRef
89 *
90 * XPLMNavRef is an iterator into the navigation database. The navigation
91 * database is essentially an array, but it is not necessarily densely
92 * populated. The only assumption you can safely make is that like-typed
93 * nav-aids are grouped together.
94 *
95 * Use XPLMNavRef to refer to a nav-aid.

161/*
162 * XPLMFindNavAid
163 *
164 * This routine provides a number of searching capabilities for the nav
165 * database. XPLMFindNavAid will search through every nav aid whose type is
166 * within inType (multiple types may be added together) and return any
167 * nav-aids found based on the following rules:
168 *
169 * If inLat and inLon are not NULL, the navaid nearest to that lat/lon will be
170 * returned, otherwise the last navaid found will be returned.
171 *
172 * If inFrequency is not NULL, then any navaids considered must match this
173 * frequency. Note that this will screen out radio beacons that do not have
174 * frequency data published (like inner markers) but not fixes and airports.
175 *
176 * If inNameFragment is not NULL, only navaids that contain the fragment in
177 * their name will be returned.
178 *
179 * If inIDFragment is not NULL, only navaids that contain the fragment in
180 * their IDs will be returned.
181 *
182 * This routine provides a simple way to do a number of useful searches:
183 *
184 * Find the nearest navaid on this frequency. Find the nearest airport. Find
185 * the VOR whose ID is "KBOS". Find the nearest airport whose name contains
186 * "Chicago".
187 *
188 */
189XPLM_API XPLMNavRef XPLMFindNavAid(
190 const char * inNameFragment, /* Can be NULL */
191 const char * inIDFragment, /* Can be NULL */
192 float * inLat, /* Can be NULL */
193 float * inLon, /* Can be NULL */
194 int * inFrequency, /* Can be NULL */
195 XPLMNavType inType);

197/*
198 * XPLMGetNavAidInfo
199 *
200 * This routine returns information about a navaid. Any non-null field is
201 * filled out with information if it is available.
202 *
203 * Frequencies are in the nav.dat convention as described in the X-Plane nav
204 * database FAQ: NDB frequencies are exact, all others are multiplied by 100.
205 *
206 * The buffer for IDs should be at least 6 chars and the buffer for names
207 * should be at least 41 chars, but since these values are likely to go up, I
208 * recommend passing at least 32 chars for IDs and 256 chars for names when
209 * possible.
210 *
211 * The outReg parameter tells if the navaid is within the local "region" of
212 * loaded DSFs. (This information may not be particularly useful to plugins.)
213 * The parameter is a single byte value 1 for true or 0 for false, not a C
214 * string.
215 *
216 */

232/*
233 * Note: the FMS works based on an array of entries. Indices into the array
234 * are zero-based. Each entry is a nav-aid plus an altitude. The FMS tracks
235 * the currently displayed entry and the entry that it is flying to.
236 *
237 * The FMS must be programmed with contiguous entries, so clearing an entry at
238 * the end shortens the effective flight plan. There is a max of 100
239 * waypoints in the flight plan.
240 *
241 */
